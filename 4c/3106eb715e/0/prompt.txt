# Plan: Pass sender name to LLM for all channel types

## Context

GitHub issue #713: When Moltis receives messages from Telegram (or Discord, WhatsApp, etc.),
the LLM cannot identify who sent the message. All channel handlers already extract `sender_name`
and `username` into `ChannelMessageMeta`, and this metadata is persisted in the session JSON.
However, `ChatMessage::User` has no `name` field, so the identity is silently dropped when
converting persisted messages to typed `ChatMessage`s for the LLM.

## Approach

Add an optional `name` field to `ChatMessage::User` and thread it through both conversion paths:
1. **History path**: `values_to_chat_messages()` extracts `sender_name`/`username` from persisted
   channel metadata.
2. **Current message path**: `run_streaming()`/`run_with_tools()` receive the sender name from
   `params["channel"]` and set it on the new user message.

The OpenAI Chat Completions API natively supports `"name"` on user messages.
Anthropic doesn't have a `name` field, so we prepend `[sender_name]: ` to the content text.

## Files to modify

### 1. `crates/agents/src/model.rs` — Core type + serialization

**a) Add `name` field to `ChatMessage::User` (line 51):**
```rust
User {
    content: UserContent,
    name: Option<String>,
},
```

**b) Update convenience constructors `user()` and `user_multimodal()` (lines 94-105):**
Keep existing constructors with `name: None` for backward compatibility.
Add new constructors:
```rust
pub fn user_named(content: impl Into<String>, name: impl Into<String>) -> Self { ... }
pub fn user_multimodal_named(parts: Vec<ContentPart>, name: impl Into<String>) -> Self { ... }
```

**c) Update `to_openai_value()` (line 141-163):**
Include `"name"` in the JSON when `name.is_some()`:
```rust
ChatMessage::User { content, name } => {
    let mut msg = match content { ... };
    if let Some(n) = name { msg["name"] = serde_json::Value::String(n.clone()); }
    msg
}
```

**d) Update `values_to_chat_messages()` (line 235):**
Extract sender name from the persisted `channel` field:
```rust
"user" => {
    let sender_name = val.get("channel")
        .and_then(|ch| ch["sender_name"].as_str()
            .or_else(|| ch["username"].as_str()))
        .map(|s| s.to_string());
    // ... existing content handling ...
    // Set name on created ChatMessage
}
```

### 2. `crates/providers/src/anthropic.rs` — Anthropic format (lines 396-419)

Prepend `[name]: ` to user content text when name is present:
```rust
ChatMessage::User { content, name } => {
    let prefix = name.as_ref().map(|n| format!("[{n}]: ")).unwrap_or_default();
    match content {
        UserContent::Text(text) => {
            let content_str = format!("{prefix}{text}");
            out.push(json!({"role": "user", "content": content_str}));
        },
        // Similar for multimodal — prepend to first text block
    }
}
```

## Out of scope

- **Web chat sender identity**: Web chat doesn't use `ChannelMessageMeta`. Supporting username
  display for web sessions would require auth context, which is a separate concern.
- **`genai` crate name support**: The `genai` crate abstraction may not support `name`.
  We'll ignore it for now (`name: _`).
